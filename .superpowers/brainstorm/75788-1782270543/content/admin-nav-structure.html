Where does compensation monitoring live in the admin?

We need to place personal BV, group BV (L/R), daily cut-offs, weekly payouts, carry-forwards, wallets, and manual override controls. Three structural options:

A

New "Compensation" section in admin sidebar

A dedicated top-level sidebar section with sub-pages: GSB Overview, Daily Cut-offs, Weekly Payouts, Carry-forwards, Wallet Ledger, Manual Controls. The existing distributor profile page gets a "Compensation →" quick-link that jumps to their data within this section.

Pros

  • All compensation data in one place
  • Easy to build reports across all distributors
  • Manual controls isolated from identity/KYC admin

Cons

  • Admin must navigate away from distributor profile to see their comp data
  • More sidebar links to manage
B

Extra tabs on the existing distributor profile page

The distributor show page (which already has identity, KYC, tree, BV ledger links) gets new tabs: Personal BV | Group BV | Wallet | GSB History | Payouts | Overrides. All compensation data is per-distributor, inline.

Pros

  • Everything about a distributor in one page
  • No context switch when investigating a specific account
  • Follows existing pattern (BV Ledger is already a linked sub-page)

Cons

  • No cross-distributor aggregate view (e.g. "show all failed cut-offs today")
  • Distributor profile page gets very busy
C

Both: new sidebar section + per-distributor deep-link (recommended)

A "Compensation" sidebar section for aggregate views (all daily cut-offs, all pending payouts, failed jobs, company-wide GSB pool). The distributor profile page gets a "Compensation →" tab/button that deep-links into that distributor's slice within the compensation section.

Pros

  • Aggregate monitoring (failed cut-offs, payout batches) in one place
  • Per-distributor drill-down from profile page
  • Scales as more comp features are added (ranks, fortune bonus, etc.)
  • Mirrors how BV Ledger works today (global list + per-distributor show)

Cons

  • More pages to build
  • Slightly more navigation steps to reach one distributor's data
